# Kit comercial - Zow Ventas-Almacen

Este documento sirve para presentar, probar e instalar Zow Ventas-Almacen en empresas pequeñas o medianas con un solo punto de atencion.

## Propuesta simple para explicar al cliente

Zow Ventas-Almacen es un sistema web para vender, cobrar, controlar caja, administrar productos, revisar stock, registrar compras y ver reportes desde cualquier dispositivo autorizado.

La idea principal:

- El cajero vende rapido.
- Almacen controla productos y stock.
- El encargado configura usuarios y revisa resultados.
- La empresa evita perder ventas, stock y dinero por controles manuales.

## Clientes ideales

- Tiendas de barrio con crecimiento.
- Minimarkets.
- Ferreterias pequenas.
- Farmacias sin facturacion integrada.
- Tiendas de ropa.
- Venta de repuestos.
- Distribuidoras pequenas con un solo punto de atencion.
- Negocios familiares donde 1 o 2 personas hacen todo.

## Beneficios que se deben comunicar

- Control de ventas del dia.
- Control de caja por cajero.
- Stock actualizado automaticamente.
- Alertas de productos bajos o por vencer.
- Compras y proveedores ordenados.
- Reporte de utilidad por producto.
- Usuarios con permisos por rol.
- Sistema en la nube, sin instalar programas pesados.
- Puede adaptarse a tiendas pequenas con 1 o 2 usuarios.

## Objeciones comunes y respuesta

| Objecion del cliente | Respuesta recomendada |
| --- | --- |
| Yo vendo en cuaderno | El sistema no elimina tu forma de trabajar de golpe; primero ordena ventas, stock y caja para que tengas control diario. |
| Mi tienda es pequena | Justamente se puede configurar para 1 o 2 personas con permisos amplios. Luego crece contigo. |
| No soy bueno con computadoras | La pantalla de caja esta pensada para tocar productos, cobrar y emitir comprobante con pocos pasos. |
| Me preocupa perder datos | Los datos quedan guardados en la nube y separados por empresa. |
| Tengo varios precios | Se puede manejar precio base y promociones. Para listas avanzadas se puede ampliar despues. |
| No quiero que todos vean ganancias | Solo encargado, supervisor u operador autorizado pueden ver utilidad y reportes sensibles. |

## Checklist de instalacion para cliente

### Antes de crear la empresa

- Confirmar nombre comercial.
- Confirmar nombre del encargado.
- Confirmar celular y correo.
- Definir plan contratado.
- Definir fecha de inicio y vencimiento.
- Definir si usara Correspondencia, Ventas-Almacen o ambos.

### En Panel ZOW

1. Crear empresa.
2. Activar sistema Ventas-Almacen.
3. Crear credencial del encargado.
4. Configurar plan y vigencia.
5. Guardar y probar login.

### En Zow Ventas-Almacen

1. Ingresar como encargado.
2. Configurar datos de tienda.
3. Definir cantidad de cajas.
4. Crear usuarios.
5. Registrar productos iniciales.
6. Crear proveedor principal.
7. Abrir caja de prueba.
8. Registrar primera venta.
9. Revisar reportes.

### Verificacion final

- Login correcto del encargado.
- Caja abre y cobra.
- Producto descuenta stock.
- Reporte muestra venta.
- Usuario cajero no ve configuracion sensible.
- Usuario almacen no ve utilidad si no corresponde.
- Empresa no ve datos de otra empresa.

## Guion de demo de 15 minutos

### Minuto 1-2: problema

Explicar:

Muchas tiendas venden todos los dias, pero no siempre saben cuanto ganaron, que productos bajaron de stock, que cajero cobro, que venta se anulo o que mercaderia deben reponer.

### Minuto 3-5: caja

Mostrar:

1. Login del cajero.
2. Apertura de caja.
3. Venta rapida.
4. Cobro en efectivo.
5. Comprobante.

Mensaje clave:

El cajero puede vender rapido sin entrar a pantallas complicadas.

### Minuto 6-8: stock

Mostrar:

1. Producto vendido descuenta stock.
2. Alerta de stock minimo.
3. Kardex.
4. Compra de reposicion.

Mensaje clave:

El negocio sabe que tiene, que falta y que se movio.

### Minuto 9-11: control del encargado

Mostrar:

1. Reporte de ventas.
2. Utilidad por producto.
3. Auditoria.
4. Usuarios y permisos.

Mensaje clave:

El dueño controla sin estar todo el dia encima de la caja.

### Minuto 12-13: promociones

Mostrar:

1. Crear promocion.
2. Agregar producto al POS.
3. Ver descuento automatico.

Mensaje clave:

Las ofertas se aplican de forma ordenada y no dependen de memoria del cajero.

### Minuto 14-15: cierre

Preguntar:

- Cuantas personas usan caja?
- Cuantos productos manejan?
- Manejan credito a clientes?
- Necesitan controlar vencimientos?
- Quieren empezar con carga manual o importacion?

Cerrar:

Podemos crear una empresa demo con sus datos reales basicos para que prueben el flujo de venta y stock.
